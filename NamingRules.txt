Naming Rules
PascalCave (every word captialzed. no space)
	used for classes and script files
	node names in scene tree
	file names

snake_case
	used for functions and and built-in functions (like func spawn_enemy)
	variables (var move_speed = 200)
	signals

UPPER_SNAKE_CASE
	Constants

Coding rules
_ready centered coding. Signals, classes, constants and variable are written first,
then _ready
then the functions
When something new is added it should be placed nearest the _ready on the approriate side
this way you can easily find and edit the newest feature is most likely to need editing
